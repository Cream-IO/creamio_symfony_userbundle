    • Introduction

    This API is part of a user bundle for Symfony. It can be used to handle authentication to a backoffice, and manage users and permissions.

    It has been made to have a separation between backend application focusing on data management, and frontend application only doing API calls to interact with the backend application.

    You can have a look at the functionnal tests report here and the associated code coverage report here.

    User structure

    This is the structure of the user entity

    Parameter Description Type Required Constraint(s)
    id User UUID string(uuid) N/A (AutoGenerated) N/A (AutoGenerated)
    username Username string Yes Between 3 and 30 characters
    plain_password Plain password to encrypt string Yes Between 5 and 4096 characters
    email E-mail address string Yes Valid e-mail address format
    firstName First Name string Yes Between 2 and 80 characters
    lastName Last Name string Yes Between 2 and 80 characters
    roles Permissions roles array Yes No constraint
    job Job's name string No Maximum 160 characters
    description User description string No Maximum 4000 characters

    Authentication

    User authentication route

    Example payload:

    {
  "username": "MyUsername",
  "password": "MyPassword"
}

    The above command returns JSON structured like this:

    {
  "status": "success",
  "code": 200,
  "request-method": "GET",
  "results-for": "login"
}

    This endpoint handles authentication by sending back PHPSESSID cookie.

    HTTP Request

    POST http://example.com/admin/api/login

    Query Parameters

    Parameter Description Type Required
    username Username string Yes
    password Password string Yes

    HTTP Response Status Code

    Code Description
    200 OK
    405 Method Not Allowed
    401 Invalid credentials
    Remember â€” the authentication works only if the cookie is stored client-side.

    User management

    Create a Specific User

    Example payload:

    {
  "username": "TestUser",
  "plain_password": "TestPass",
  "email": "test@test.com",
  "roles": [
    "ROLE_ADMIN"
  ],
  "first_name": "Test",
  "last_name": "User",
  "job": "Developer",
  "description": "My beautiful description"
}

    Example return:

    {
  "status": "success",
  "code": 201,
  "request-method": "POST",
  "request-ressource-id": "bfb87adf-60ac-4027-a5be-6007f946791c"
}

    This endpoint create a new user.

    HTTP Request

    POSET http://example.com/admin/api/users

    Query Parameters

    Parameter Description Type Required
    username Username string Yes
    plain_password Plain password to encrypt string Yes
    email E-mail address string Yes
    firstName First Name string Yes
    lastName Last Name string Yes
    roles Permissions roles array Yes
    job Job's name string No
    description User description string No

    HTTP Response Status Code

    Code Description
    201 OK - Created
    400 Bad Request - Content type is incorrect or value(s) are missing/incorrect
    The response "Location" header is set to the details (GET) route to created user details

    Patch a Specific User

    Example payload:

    {
  "username": "MyNewUsername",
  "plain_password": "MyNewPassword"
}

    Example return:

    {
  "status": "success",
  "code": 200,
  "request-method": "PATCH",
  "request-ressource-id": "c7ac5d39-8c16-40ce-b7fb-dbbe62650a0d"
}

    This endpoint patches a specific user datas, replacing the values by the provided ones, leaving the non provided variables as is.

    HTTP Request

    PATCH http://example.com/admin/api/users/<ID>

    URL Parameters

    Parameter Description
    ID The ID of the user to patch

    Query Parameters

    Parameter Description Type Required
    username Username string No
    job Job's name string No
    description User description string No
    roles Permissions roles array No
    email E-mail address string No
    firstName First Name string No
    lastName Last Name string No
    plain_password Plain password to encrypt string No

    HTTP Response Status Code

    Code Description
    200 OK
    404 Invalid ID or non-existent user

    The API uses the following error codes:

    Error Code Meaning
    400 Bad Request -- Your request is invalid, can be due to Content-Type header not set to application/json.
    401 Unauthorized -- Your session ID in cookies is wrong, re-authenticate.
    403 Forbidden -- You are logged-in but your permissions are not sufficient for this request.
    404 Not Found -- The requested resource could not be found.
    405 Method Not Allowed -- You tried to access a route with an invalid method.
    500 Internal Server Error -- We had a problem with our server. Try again later.
    503 Service Unavailable -- We're temporarily offline for maintenance. Please try again later.
